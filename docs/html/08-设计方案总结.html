设计方案总结

企业级 RAG 知识库 — 完整技术选型与架构决策

最终决策汇总

决策项 选择 理由
应用场景 企业内部知识库 3000 人，部门级使用
架构策略 模块化单体 → 渐进微服务 150 QPS 峰值，单体足够；Maven 多模块预留拆分
主体框架 Spring Boot 3.5 + WebFlux 流式 SSE 响应，非阻塞 IO
AI 编排 LangChain4j 1.17+ @AiService / Tool / RAG 一体化
RAG 策略 高级 RAG（混合检索+Rerank） 三路召回 + RRF 融合 + BGE Reranker
LLM DeepSeek V4 → 通义千问 主备双模型，降级策略
Embedding BGE-M3（本地部署） 中英双语，1024维，稠密+稀疏双表示
向量数据库 Milvus Standalone 10 亿级，HNSW 索引，混合检索
关键词检索 Elasticsearch 8.x BM25 + 全文检索 + 高亮
会话记忆 Redis Stack 低延迟，TTL 自动淘汰，JSON 原生支持
业务数据 PostgreSQL 16 多租户、审计日志、权限管理
文档存储 MinIO (S3 兼容) 自建对象存储，零成本
消息队列 RabbitMQ 文档异步处理、事件驱动
文档解析 Apache Tika + PDFBox + Tesseract 1000+ 格式 + OCR
认证 Spring Security + OAuth2 SSO 对接 OA 系统
多租户 按部门 + 共享库 tenant_id 隔离 + 跨部门共享
缓存 Caffeine(L1) + Redis(L2) 热点问题缓存 + 穿透保护
监控 Prometheus + Grafana + Actuator 指标 + 告警 + 健康检查
